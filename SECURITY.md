# Security Policy

The Velo project takes security seriously. We welcome responsible disclosure of vulnerabilities and will make every reasonable effort to investigate and address issues promptly.

## Reporting a Vulnerability

Please do not open a public issue for security-sensitive findings. Instead, report them privately through the repository’s security reporting process.

If you believe you have found a vulnerability in the contracts, API, mobile app, or infrastructure configuration:

1. describe the issue clearly,
2. include steps to reproduce where possible,
3. avoid sharing exploit details publicly until we have had an opportunity to assess the issue.

## What to Expect

- acknowledgment within a reasonable timeframe,
- a triage and impact assessment,
- coordinated remediation where appropriate,
- public disclosure only after a fix is ready or after a reasonable resolution window.

## Scope

This policy applies to:

- Soroban contracts,
- API routes and middleware,
- frontend and mobile client code,
- deployment configuration and environment handling.

## Safe Harbor

We ask reporters to avoid privacy violations, destructive testing, or disruptive access to third-party systems. Responsible disclosure is expected.
