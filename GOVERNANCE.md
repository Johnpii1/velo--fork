# Governance

Velo is maintained by the Nullifier Systems community and a small set of project maintainers responsible for technical direction, release health, and review standards.

## Roles

- Contributors: propose changes, report issues, and submit pull requests.
- Maintainers: review contributions, merge approved work, and guide project direction.
- Core Maintainers: oversee release planning, security posture, and architectural consistency.

## Decision-Making

Routine changes are handled through pull requests and maintainer review. Larger changes that affect the contract model, security posture, or public interface should be discussed in an issue or GitHub discussion before implementation.

## Release Management

Releases should be coordinated by maintainers and documented in the changelog. Changes that affect smart contract behavior should be reviewed carefully and tested before merging.

## Conflict Resolution

Disagreements should be handled constructively and in public where possible. If consensus cannot be reached, maintainers may make a decision based on project stability, security, and long-term maintainability.
